Project Architecture & System Design

System Architecture Overview

Our healthcare management system follows a modern three-tier architecture with clear separation between presentation layer (Angular Frontend), business logic layer (Node.js Backend), and data layer (Supabase/PostgreSQL). The system integrates multiple third-party services for enhanced functionality.

System Components & Integration Points

Frontend Layer

  • Angular Components: Reusable UI components for appointments, user management
  • Services: HTTP clients for API communication and state management
  • Guards: Route protection and authentication validation
  • Interceptors: Request/response handling and error management

Backend Layer

  • API Routes: RESTful endpoints for all business operations
  • Middleware: Authentication, validation, and error handling
  • Controllers: Business logic and data processing
  • Services: External API integrations and utility functions

Data Layer

  • PostgreSQL: Primary database for structured healthcare data
  • Supabase: Real-time subscriptions and authentication
  • Migrations: Database schema versioning and updates
  • Indexes: Performance optimization for complex queries

External Integrations

  • Payment Gateway: VNPay for secure transaction processing
  • Communication: SMS/Email services for notifications
  • Storage: Cloud services for file and image management
  • AI Services: Machine learning for predictive analytics
